<task type="auto">
  <name>Task 2: Configure Tailwind CSS v4 with PostCSS and dark theme</name>
  <files>
    postcss.config.mjs
    app/globals.css
  </files>
  <action>
Install Tailwind v4 beta packages:
```bash
npm install -D tailwindcss@next @tailwindcss/postcss@next postcss@latest
```

Create `postcss.config.mjs` (NOT .js - use ES modules):
```javascript
export default {
  plugins: {
    "@tailwindcss/postcss": {},
  },
};
```

Replace contents of `app/globals.css` with Tailwind v4 syntax:
```css
@import "tailwindcss";

@theme {
  /* Dark theme color palette */
  --color-gray-950: #0a0a0b;
  --color-gray-900: #18181b;
  --color-gray-800: #27272a;
  --color-gray-700: #3f3f46;
  --color-gray-50: #fafafa;
  --color-gray-100: #f4f4f5;
}

/* Enable dark mode with .dark class */
@custom-variant dark (&:where(.dark, .dark *));
```

**Critical differences from Tailwind v3:**
- Use `@import "tailwindcss"` NOT `@tailwind base/components/utilities`
- Configuration lives in CSS via `@theme`, NOT in tailwind.config.js
- PostCSS plugin is `@tailwindcss/postcss`, NOT `tailwindcss`

**Why not keep v3:** Project requirements specify Tailwind v4, which offers 3-8x faster builds and modern CSS features. v4 is production-ready beta.
  </action>
  <verify>
Run `npm run dev` and confirm:
- No PostCSS errors in terminal
- Tailwind classes render correctly (check browser DevTools on welcome page)

Test Tailwind is working by temporarily adding `className="bg-gray-950 text-gray-50 p-4"` to a component and verifying styles apply in browser.
  </verify>
  <done>
Tailwind v4 processes CSS without errors, Tailwind utility classes apply correctly when used in components, and dark theme colors are available via custom theme variables.
  </done>
</task>
